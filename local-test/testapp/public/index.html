End-to-end backup & restore verification

Idle — never run

Wipes both databases, seeds a known dataset (state A), triggers a full backup, mutates the data (state B), triggers an incremental, then verifies Restore to Latest reproduces state B and Restore to Point reproduces state A. Compares contents via SHA-256 fingerprints of every collection. Also asserts that the excluded not_a_sensor collection is correctly absent from restored data.

Idle — never run

Configurable volume test: seeds N sensor collections × M docs each, then triggers a backup (and optionally a full restore + count verification). Use this to measure end-to-end timing and resource usage at scale. Verification is count-based here — fingerprint hashing 250M docs would itself take many minutes.
